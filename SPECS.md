# Project Chaos-1B: Automated Red Teaming via Asynchronous RFT

## 1. Abstract
This project implements an automated, iterative Red Teaming loop to evaluate the robustness of aligned LLMs against adversarial attacks generated by smaller, weaker models. By utilizing Rejection Sampling Fine-Tuning (RFT), we demonstrate that a 1B parameter model can learn to reliably jailbreak a 3B parameter model through iterative self-play, without requiring human-in-the-loop optimization.

## 2. Technical Architecture
The system is designed for constrained hardware (Apple Silicon, 16GB Unified Memory) by enforcing strict asynchronous execution of three distinct agents:

1.  **The Adversary (Generator):** `Llama-3.2-1B-Instruct` (4-bit). 
    * *Role:* Generates adversarial prompts to solicit harmful content.
    * *Optimization:* Fine-tuned via LoRA on successful attack trajectories.
2.  **The Victim (Target):** `Llama-3.2-3B-Instruct` (4-bit).
    * *Role:* The system under test. Represents a standard "aligned" model.
    * *State:* Frozen parameters.
3.  **The Judge (Evaluator):** `Llama-Guard-3-1B-INT4`.
    * *Role:* Binary classification of Victim responses (Safe/Unsafe).
    * *State:* Frozen parameters.

## 3. Operational Logic (The Chaos Loop)
The system operates in discrete rounds ($R_0 \to R_N$):
1.  **Generation Phase:** Adversary generates $K$ candidate attacks for a specific harmful intent.
2.  **Evaluation Phase:** Victim processes candidates; output is captured.
3.  **Adjudication Phase:** Judge classifies Victim outputs.
4.  **Learning Phase:**
    * If $Score(Attack) == Unsafe$ (Success), the attack is added to the training set.
    * Adversary performs $N$ steps of QLoRA training on the curated dataset.
    * *Hypothesis:* The Adversary will converge on high-probability jailbreak vectors (e.g., persona adoption, code injection).

## 4. Hardware Constraints & Mitigations
* **Constraint:** 16GB RAM prevents simultaneous loading of multiple LLMs.
* **Mitigation:** The pipeline utilizes Python `subprocess` calls to spin up and tear down distinct processes for each phase, ensuring complete VRAM flushing between steps.

## 5. Goals
* **Baseline:** Establish a "Success Rate" curve over 10 rounds.
* **Artifact:** Produce a set of LoRA adapters that turn a generic 1B model into a specialized Red Teamer.
